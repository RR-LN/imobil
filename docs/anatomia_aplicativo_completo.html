A lógica de negócio é o cérebro da app — as regras que transformam acções em resultados.

Tipos de lógica
Validação — verificar se os dados estão correctos antes de agir
Transformação — converter dados de um formato para outro
Cálculo — operações matemáticas (preços, comissões, datas)
Autorização — verificar se o utilizador pode fazer a acção
Orquestração — coordenar múltiplas operações em sequência
Notificação — decidir quando e quem notificar
Onde corre cada regra
Só no servidor — cálculo de comissões, geração de códigos, preços
No servidor e cliente — validação de formulários (UX no cliente, segurança no servidor)
Só no cliente — formatação de datas, ordenação de listas, animações
Regra de ouro — nunca confiar só no cliente para dados com impacto financeiro
Exemplos reais no KuGava
commission.ts
Calcula quanto o afiliado ganha: 2% para agentes, 0.5% ou 500 MZN para utilizadores
referralHandler.ts
Detecta link de afiliado ao abrir a app, guarda código no AsyncStorage até ao registo
updateAffiliateLevel()
Bronze→Prata→Ouro→Platina conforme número de conversões acumuladas

Segurança funciona em camadas — cada uma cobre o que a anterior pode deixar passar.

1
Autenticação — quem és tu?
JWT tokens assinados pelo Supabase. Expiram em 1h e são renovados automaticamente. Sem token válido = sem acesso.
2
Autorização — o que podes fazer?
RLS policies no PostgreSQL. Cada query é filtrada: só vês os teus dados, as tuas mensagens, os teus imóveis.
3
Validação — os dados são válidos?
Validar no cliente (UX) E no servidor (segurança). Tipos, formatos, limites, conteúdo injectado.
4
Encriptação — os dados estão protegidos?
HTTPS para tudo em trânsito. Passwords com bcrypt (Supabase). Dados sensíveis nunca em logs.
5
Rate limiting — estás a abusar?
Limite de pedidos por IP e por utilizador. Previne spam de registos, força bruta em passwords, abuso do sistema de afiliados.
