Crítico — vai quebrar com volume
CRÍTICO
Sem índices nas tabelas
properties, messages e bookings não têm índices. Com 10k imóveis, cada pesquisa faz full table scan — latência de 2-8 segundos.
Fix: CREATE INDEX idx_properties_city ON properties(city); CREATE INDEX idx_properties_status ON properties(status); CREATE INDEX idx_messages_conv ON messages(conversation_id, created_at DESC);
CRÍTICO
Pesquisa sem full-text search
searchProperties() usa LIKE '%query%' — não usa índices e é lento. Com 50k imóveis = 400ms+ por pesquisa.
Fix: Adicionar coluna search_vector tsvector, usar to_tsquery() e índice GIN. Ou activar pgvector para pesquisa semântica.
CRÍTICO
Sem paginação real
getProperties() busca todos os registos. Com 5k imóveis = 15MB de JSON na memória do telefone.
Fix: Supabase .range(from, to) — cursor-based pagination. Carregar 20 imóveis por vez com FlatList onEndReached.
Importante — degradação progressiva
IMPORTANTE
N+1 queries no feed
HomeFeed busca imóveis, depois busca o agente de cada um separadamente. 20 imóveis = 21 queries.
Fix: JOIN com profiles na mesma query: .select('*, profiles(full_name, avatar_url)')
IMPORTANTE
Sem cache local
Cada vez que o utilizador volta ao feed, faz nova chamada à API. Consome dados móveis e é lento em redes 3G (Moçambique).
Fix: react-query ou SWR para cache com stale-while-revalidate. Dados ficam 5 min em cache.

Optimizações no código
IMPORTANTE
Canal subscrito mas não limpo
Se o utilizador navega entre conversas rapidamente, cria múltiplos canais sem fazer unsubscribe dos anteriores. Memory leak + conexões fantasma.
Fix: Guardar referência do canal em useRef e garantir cleanup no useEffect return.

Subscrito a conversas globais

Problemas de storage
CRÍTICO
Sem compressão de imagens
expo-image-picker devolve imagens originais — podem ter 4-8MB cada. 6 fotos por imóvel × 1000 imóveis = 24-48GB. Esgota storage em dias.
Fix: Usar ImageManipulator do Expo antes do upload: comprimir para quality: 0.7 e resize para maxWidth: 1200. Reduz 80% do tamanho.
CRÍTICO
Sem CDN para imagens
Imagens servidas directamente do Supabase Storage sem CDN. Utilizadores em Nampula ou Beira descarregam de servidores distantes — 3-8 segundos de loading.
Fix: Activar Supabase CDN (incluído no Pro) ou usar Cloudflare Images ($5/mês). URLs ficam com cache na edge próxima do utilizador.
IMPORTANTE
Sem thumbnails automáticos
O feed carrega a imagem original (1-4MB) para mostrar num card de 120px. Desperdício de dados móveis — crítico em Moçambique com planos de dados caros.
Fix: Supabase Image Transformations: adicionar ?width=300&quality=60 à URL. Gera thumbnail on-the-fly.
JÁ OK
Bucket separado por tipo
Estrutura properties/ já separada. Boa base para gerir permissões e limpeza por tipo de conteúdo.
Performance no dispositivo
CRÍTICO
FlatList sem optimização
HomeFeed renderiza todos os cards sem getItemLayout, windowSize ou maxToRenderPerBatch. Em dispositivos Android de gama baixa (maioria em Moçambique), a scroll fica aos saltos.
Fix: Adicionar getItemLayout (altura fixa por item), windowSize={5}, maxToRenderPerBatch={5}, removeClippedSubviews={true}.
IMPORTANTE
Imagens sem lazy loading
Image nativo do React Native não tem cache persistente. Cada vez que o utilizador volta ao feed, as imagens descarregam novamente.
Fix: Instalar expo-image (substituto moderno com cache automático e placeholders blur). 3 linhas de mudança.
IMPORTANTE
Bundle size sem optimização
Sem tree-shaking configurado, o bundle inclui código não usado. Tempo de arranque mais lento, especialmente em Android.
Fix: Activar Hermes engine no app.json (já disponível no Expo). Reduz bundle e melhora 30-50% o tempo de arranque.
JÁ OK
Zustand para state
Zustand é leve e não causa re-renders desnecessários. Boa escolha para escala. Sem problemas aqui.
Offline & conectividade
MELHORAR
Sem suporte offline
Em Moçambique a internet é instável. Se o utilizador perde conexão, a app fica em branco ou mostra erros sem feedback. Para 10k+ utilizadores com redes 2G/3G, isso é um problema real.
Fix progressivo: 1. Detectar NetInfo e mostrar banner "Sem internet". 2. Guardar últimos imóveis vistos no AsyncStorage. 3. Queue de mensagens offline que enviam quando a conexão volta.
Vulnerabilidades actuais
CRÍTICO
RLS policies incompletas
Se as policies não cobrem todos os casos, qualquer utilizador autenticado pode ler mensagens de outros, ver dados de afiliados alheios ou modificar imóveis que não são seus.
Fix: Auditar cada tabela: profiles (só o próprio), properties (owner ou public se active), messages (só participantes da conversa), affiliates (só o próprio), bookings (comprador ou vendedor).
CRÍTICO
Sem validação no servidor
createProperty() confia nos dados do cliente. Alguém pode publicar um imóvel com preço negativo, tipo inválido, ou injectar código no título/descrição.
Fix: Criar Edge Function validate-property com zod ou yup para validar antes de inserir. Nunca confiar no cliente para dados críticos.
IMPORTANTE
Referral code previsível
Código gerado como 'KG' + nome + número random. Fácil de adivinhar e potencialmente manipulável para inflar comissões.
Fix: Gerar código no servidor (Edge Function) com crypto.randomUUID() truncado. Nunca no cliente.
IMPORTANTE
Sem rate limiting
Sem limite de pedidos, alguém pode fazer 1000 registos falsos via link de afiliado para gerar bónus fraudulentos.
Fix: Activar Supabase Rate Limiting nas Edge Functions. Limite de 5 registos por IP por hora.
O que falta para o produto ser competitivo
PRODUTO
Verificação de imóveis
Sem verificação, qualquer um publica imóveis falsos ou duplicados. Destrói a confiança rapidamente.
Fix: Badge "Verificado" por agentes PRO. Reportar anúncio. Moderação manual dos primeiros 500.
PRODUTO
Sem avaliações
Compradores não sabem se o agente é de confiança. Isso trava o primeiro contacto.
Fix: Rating de 1-5 estrelas após visita confirmada. Mostra média no perfil do agente.
PRODUTO
Sem mapa
Utilizadores querem ver imóveis num mapa, não só numa lista. Feature mais pedida em marketplaces imobiliários.
Fix v1.1: react-native-maps com pins de imóveis. Filtro por raio de distância.
ESCALA
Analytics em falta
Sem dados, não sabes o que funciona. Qual imóvel tem mais cliques? Qual cidade converte mais?
Fix: Mixpanel free tier ou PostHog self-hosted. Eventos: view_property, start_chat, book_visit, share_referral.
ESCALA
Sem suporte ao utilizador
Com 10k utilizadores vão surgir disputas, fraudes e problemas técnicos. Sem canal de suporte = reviews negativas na Play Store.
Fix: Botão "Reportar problema" no perfil → formulário → email. Intercom free tier para chat de suporte.
